Representing computation as graphs in state machine.

In event based systems the sequence of computation is determined by the sequence
of events. The state of the system is the combination of the existing state and
the information from the event.

Between the extremes of the generalization of =next-state = current-state ×
event= and an exhaustive enumeration of every possible permutation there is a
space of useful abstractions: information independent of its specific
representation.





Commonly the events are asynchronous and the state of the system has
to be kept and restored between events.
